[数学]思路

Posted on 2019-11-01 | In 数据结构

我记得我高中也有几个本子专门记录解题思路的，效果貌似还可以.(数学从70补到130，理综从150到了250)。就用这个方法在ACM上试试.

思路就是题目与方法间的link而已.

1、快速求出n！的素因数的指数，也就是对n！进行素数分解。结合这个，就可以快速求出大组合数的素数分解。

2、F[i]=F[i-2]*2+F[i-1]+i^4 这种递推式可以用矩阵来加速递推.

3、对于a^b (mod m) 这种式子，如果b特别大，huge mod，那么我们可以考虑使用欧拉降幂公式 A^K≡A^(K%ϕ(m)+ϕ(m))( mod m) (K>ϕ(m))

证明 https://blog.csdn.net/weixin_38686780/article/details/81272848
4、枚举一个n的所有因子，可以考虑枚举所有小于sqrt(n)的因子，时间复杂度是o(sqrt(n)).

5、如果题目给的限制条件很多，并且相互都独立，可以考虑用容斥定理UVA - 11401

6、两个多项式的GCD，思路和普通的gcd一样，只不过要自己重载一下乘除法.

7、置换的性质：任何一个置换都可以表示成若干个对换的乘积，且表示形式有无穷种，但是不同的形式，其奇偶型始终保持一致。

8、置换转换成对换的乘积：如果存在一个置换为(a b c d) 则他可以转换为(a b) (a c) (a d)的形式。所以k阶的循环置换，可以转换为k - 1个对换的乘积(当然只是其中一种形式).

9、找循环置换阶数的方法：将置换以有向图的方式保存，然后通过DFS求有向圈的长度即可。

10、求a的逆元（模b），如果a和b是互质的，用扩欧求逆元。扩欧解的是ax + by = gcd(a,b)= 1这个方程，即ax = 1(mod b) 所以x就是a的逆元(模b)。通常情况下，a和b互质往往是这二者是一个数的素因子.

11、将询问离线：如果题目刻意告诉你 t组数据，每组数据都有一个n，但是这t个n中 最多只有x个不同的n，也就是说，测试组合很多，但是有许多相同的n。那么我们可以进行离线操作，将询问保存下来，排个序，将n相同的几组测试数据放在一起测试，省的重新建图之类的。

12、有些特别大的数，并且要对这些特别大的数进行一些繁琐的操作（例如快速幂之类的）显然要你自己写大数，并且在实现大数的乘法的时候，可以考虑用FFT来优化。

13、题目要将n进行素因数分解，我们只需要预处理出[2…sqrt(n)]的素数即可.因为如果n的因子包括sqrt(n)和某个大于sqrt(n)的数的话，他们的乘积就已经大于n了.

14、使用高精度时候，可以结合压位操作来优化乘法常数。

15、要计算大数的幂，乘等操作时，可以用FFT将卷积的O(n^2) 优化成O(nlogn).

16、估计超大数的对数，可以从他的十进制长度着手开始估计.例：n的十进制长度为1e6以下，是一个很大的大数了。那么我们怎么预估它的log3呢。

我们可以知道 n >= 10^k (k为某个数)。因此，log3(n) >= k * log3(10);

17、求n!中含有某个因子个数的方法 ：

N!的质因子x的个数 == [N/x]+[N/x^2]+[N/x^3]+….

17.2、将1….n中的n个数进行质因子分解，那么这n个数分解后的素数p的指数之和，也可以用以上方式求.

18、N!的二进制表示中最低位1的位置 == 求 N!的质因子2的个数

19 、N!的质因子2的个数的另一个方法：
N!含有质因数2的个数，等于N减去N的二进制表示中1的个数

20、模数为素数时，求逆元的时候尽量使用费马小定理.(方便快捷.)

21、整除2的操作，可以等价为>>，乘2的操作，可以理解为<<

22、在一个序列中，寻找规定长短的区间，使得其中的某个属性值为k最优，每个点对于它所在的区间的属性有贡献。那么我们可以用滑动窗口法遍历一遍序列即可。

23、Dilworth定理: 偏序集能划分成的最少的全序集个数等于最大反链的元素个数

Dilworth定理的对偶定理：对于一个偏序集，其最少反链划分数等于其最长链的长度。

24、要使得若干个数的乘积为n，且使得这些数的和最小，那么这些数中3的数目一定越多越好.

25、所有大于3的素数模6或同余1，或同余5。这样就可以在条件苛刻的时候压缩筛素数的时间。

//显然压空间可以用bitset，bitset存1e8的空间只需要12.5M。再利用2,3以外的素数==6x+-1，压缩一下就搞定了。 以下代码筛3e8的素数只需要250ms。
void prime()
{
    add(2); add(3); //单独考虑 
    for(int i = 5, d = 2; i <= N; i += d, d = 6 - d) {
       if(!p[i/3]){
          add(i);
          if(i > N/i) continue;
          for(int j = i * i, v = d; j <= N; j += i*v, v = 6 - v)
            p[j/3] = 1;
       } 
    }
}

26、 long long 的移位运算不能写成(1<<(h-1))，要写成（1LL<<（h-1））

27、生成树的数目： 矩阵树定理用于计算无向图生成树个数，和基尔霍夫矩阵的行列式密切相关。
