7、置换的性质：任何一个置换都可以表示成若干个对换的乘积，且表示形式有无穷种，但是不同的形式，其奇偶型始终保持一致。

8、置换转换成对换的乘积：如果存在一个置换为(a b c d) 则他可以转换为(a b) (a c) (a d)的形式。所以k阶的循环置换，可以转换为k - 1个对换的乘积(当然只是其中一种形式).

9、找循环置换阶数的方法：将置换以有向图的方式保存，然后通过DFS求有向圈的长度即可。

10、求a的逆元（模b），如果a和b是互质的，用扩欧求逆元。扩欧解的是ax + by = gcd(a,b)= 1这个方程，即ax = 1(mod b) 所以x就是a的逆元(模b)。通常情况下，a和b互质往往是这二者是一个数的素因子.

11、将询问离线：如果题目刻意告诉你 t组数据，每组数据都有一个n，但是这t个n中 最多只有x个不同的n，也就是说，测试组合很多，但是有许多相同的n。那么我们可以进行离线操作，将询问保存下来，排个序，将n相同的几组测试数据放在一起测试，省的重新建图之类的。

12、有些特别大的数，并且要对这些特别大的数进行一些繁琐的操作（例如快速幂之类的）显然要你自己写大数，并且在实现大数的乘法的时候，可以考虑用FFT来优化。

我们可以知道 n >= 10^k (k为某个数)。因此，log3(n) >= k * log3(10);

void prime()
{
    add(2); add(3); //单独考虑 
    for(int i = 5, d = 2; i <= N; i += d, d = 6 - d) {
       if(!p[i/3]){
          add(i);
          if(i > N/i) continue;
          for(int j = i * i, v = d; j <= N; j += i*v, v = 6 - v)
            p[j/3] = 1;

    }
}

dp[i][j]表示用前i个字符组成的以j为结尾的01串个数。

如果第i个字符为0，则dp[i][0] = dp[i-1][1] + dp[i-1][0] + 1，dp[i][1] = dp[i-1][1]

如果第i个字符为1，则dp[i][1] = dp[i-1][1] + dp[i-1][0] + 1，dp[i][0] = dp[i-1][0]

对于序列 1 0 1 0

dp[3][1] 表示用前i个元素组成的以1结尾的01串的个数.

dp[3][0] 表示用前i个元素组成的以0结尾的01串的个数.

显然后者dp[3][0]的数目和用前i - 1个元素组成的以0结尾的01串个数是一样.

然后前者，因为第i个元素是也可用来组成以1结尾的子串的。将所有dp[i-1][0]所记录的子串后面添加一个1，就是新的以1结尾的子串了，因此我们

dp[3][1] = dp[2][0] + dp[2][1] + 1;

POJ 3260 The Fewest Coins

Posted on 2019-10-21 | In 数据结构

Time Limit: 2000MS

Memory Limit: 65536K

Total Submissions: 8495

Accepted: 2688

Description

Farmer John has gone to town to buy some farm supplies. Being a very efficient man, he always pays for his goods in such a way that the smallest number of coins changes hands, i.e., the number of coins he uses to pay plus the number of coins he receives in change is minimized. Help him to determine what this minimum number is.

FJ wants to buy T (1 ≤ T ≤ 10,000) cents of supplies. The currency system has N (1 ≤ N ≤ 100) different coins, with values V1, V2, …, VN (1 ≤ Vi ≤ 120). Farmer John is carrying C1 coins of value V1, C2 coins of value V2, …., and CN coins of value VN (0 ≤ Ci ≤ 10,000). The shopkeeper has an unlimited supply of all the coins, and always makes change in the most efficient manner (although Farmer John must be sure to pay in a way that makes it possible to make the correct change).

InputLine 1: Two space-separated integers: N and T.
Line 2: N space-separated integers, respectively V1, V2, …, VN coins (V1, …VN)
Line 3: N space-separated integers, respectively C1, C2, …, CN

OutputLine 1: A line containing a single integer, the minimum number of coins involved in a payment and change-making. If it is impossible for Farmer John to pay and receive exact change, output -1.

Sample Input

3 70
5 25 50
5 2 1

Sample Output

3

HintFarmer John pays 75 cents using a 50 cents and a 25 cents coin, and receives a 5 cents coin in change, for a total of 3 coins used in the transaction.

                    used[j] = used[j - node[i].v] + 1;
                }
        }
        //预处理出dp2[1] ... dp2[EXTRA]
        memset(dp2, INF, sizeof dp2);
        dp2[0] = 0;
        for(int i = 1; i <= n; i++)
            for(int j = node[i].v; j <= EXTRA; j++)
                dp2[j] = min(dp2[j], dp2[j - node[i].v] + 1);
        int ans = INF;
        for(int i = 0; i <= EXTRA; i++)
            if(dp1[t + i] < INF && dp2[i] < INF)
                ans = min(ans, dp1[t + i] + dp2[i]);
        printf("%d\n", ans == INF? -1:ans);
    }
